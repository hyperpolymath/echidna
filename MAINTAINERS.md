# Maintainers

This document lists the maintainers of the ECHIDNA project.

## Current Maintainers

### Core Team

- **ECHIDNA Project Team**
  - Email: maintainers@echidna-project.org
  - Role: Core development, architecture, and project direction
  - GitLab: [@echidna-team](https://gitlab.com/echidna-team)

## Emeritus Maintainers

No emeritus maintainers at this time.

## Maintainer Responsibilities

Maintainers are responsible for:

1. **Code Review**: Reviewing and approving merge requests
2. **Issue Triage**: Labeling, prioritizing, and assigning issues
3. **Architecture**: Making decisions about project architecture and design
4. **Releases**: Managing releases and version bumps
5. **Community**: Fostering a welcoming and inclusive community
6. **Documentation**: Ensuring documentation is accurate and up-to-date
7. **Security**: Responding to security issues promptly
8. **Compliance**: Maintaining RSR/CCCP compliance

## Becoming a Maintainer

Maintainership is granted based on:

- Consistent, high-quality contributions
- Deep understanding of the project
- Commitment to the project's values and goals
- Community involvement and mentorship

If you're interested in becoming a maintainer, please reach out to the core team.

## Decision-Making Process

ECHIDNA follows a consensus-based decision-making process:

1. **Proposals**: Anyone can propose changes via issues or MRs
2. **Discussion**: Open discussion on technical merits
3. **Consensus**: Maintainers work toward consensus
4. **Decision**: If consensus cannot be reached, a vote may be called
5. **Documentation**: All decisions are documented in issues/MRs

## Code of Conduct

All maintainers must adhere to the project's [Code of Conduct](CODE_OF_CONDUCT.md).
